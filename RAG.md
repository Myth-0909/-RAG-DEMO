# RAG 智能问答系统

## 什么是 RAG

RAG（检索增强生成，Retrieval-Augmented Generation）的核心思路：大模型的知识受限于训练数据，无法回答企业内部文档、专业资料等私有知识。RAG 的做法是——用户提问时，先从知识库中检索相关内容，再把检索结果和问题一起交给大模型，让它「参考材料回答」，从而给出有据可查的答案，而不是凭空编造。

本系统基于 **FastAPI + Milvus 向量数据库 + Qwen3-Embedding-8B 嵌入模型** 构建，支持可动态切换的 LLM 后端，实现了从文档上传、智能分块、向量检索到多轮对话问答的完整链路。

## 系统流程

```
上传文档 → 内容提取 → 清洗评估 → 分块分析 → 执行分块 → 向量化存储
                                                    ↓
用户提问 → 向量检索 + 记忆召回 → LLM 生成回答 ← 对话历史
```

## 文档处理

### 1. 内容提取

支持 PDF、Word（.docx）、Markdown、TXT 四种格式。上传后系统自动提取文本内容：

- **PDF**：pypdf 提取正文 + pdfplumber 提取表格并转为 Markdown + 内嵌图片 OCR 识别（pytesseract，中英双语）
- **Word**：python-docx 提取段落和表格 + 内嵌图片 OCR
- **Markdown / TXT**：直接读取文本

### 2. LLM 驱动的清洗与分块分析

这是本系统区别于简单 RAG 方案的核心设计。大模型单次能处理的文本量有限（本系统控制在 2000 token 以内），需要将长文档拆分为多个「块」。但不同文档的结构差异很大——技术手册层级清晰，叙事报告段落连贯，对话记录结构松散——固定策略无法兼顾。

系统采用 **LLM 自动分析 + 规则清洗** 的双轨处理方式：

**清洗阶段**：将文档前 6000 字符发送给 LLM 评估（如检测 PDF 转换产生的乱码、多余空白、页码残留等），LLM 返回是否需要清洗及具体操作。同时本地规则引擎执行 Unicode 归一化、PDF 伪影移除、空白行合并等操作。LLM 分析最多重试 3 次（间隔 3s/6s），失败则回退到纯规则判断。

**分块分析阶段**：将清洗后的文本再次发送给 LLM，分析文档结构特征（层级深度、内容类型、叙述性强弱），推荐最优分块策略和参数（块大小 200-1500 字符，重叠 0-块大小/2）。同样支持 3 次重试和规则回退。

整个过程通过 **SSE 流式推送**到前端，用户可实时看到 LLM 的思考过程和每步结果。

### 3. 五种分块策略

| 策略 | 适用场景 | 原理 |
|------|----------|------|
| 固定分块 | 简单纯文本 | 固定窗口滑动切分 |
| 递归分块 | 通用文本 | 按段落→换行→句号→感叹号→分号→空格逐级降级切分 |
| 父子分块 | 层级清晰的文档 | 小块精准匹配 + 大块（4倍大小）提供完整上下文 |
| 语义分块 | 叙述性长文 | 小块切分后向量聚类，语义相近的相邻块合并（相似度阈值 0.7） |
| 混合分块 | 混合结构文档 | 按章节分割后每个章节独立判断，选用最合适的子策略 |

所有策略均包含表格保护机制：自动识别 Markdown 格式的表格，保持表格完整不被截断。

### 4. 向量化存储

分块后的文本通过 Qwen3-Embedding-8B 模型转换为 1024 维向量，批量写入 Milvus 向量数据库（每批 50 条）。向量语义相近即内容相关，检索时通过余弦相似度计算匹配度。

Milvus 索引配置：IVF_FLAT 算法，COSINE 相似度度量，nlist=128 聚类中心，检索时 nprobe=16。

## 问答流程

### 1. 检索

用户提问后，系统并行执行两条检索路径：

- **知识库检索**：将问题向量化，在选中的知识库对应 Milvus collection（`kb_{id}`）中独立搜索 top 5 结果，跨库合并后按相似度排序取全局最优
- **长期记忆召回**：计算问题向量与用户所有历史对话摘要的余弦相似度，召回最相关的 3 条历史对话

### 2. 生成

将检索结果组装为上下文，按优先级排列：
1. 长期记忆（相关历史对话摘要）
2. 知识库检索结果（带编号 [1] [2] …，优先展示父块文本以提供完整上下文）
3. 最近 5 轮对话历史（10 条消息）

LLM 在 system prompt 的约束下生成回答——仅基于上下文回答、不知则明确告知、引用来源编号。答案通过 SSE 逐字流式输出，减少等待感。每个回答附带引用来源（文档名 + 匹配度百分比），支持溯源核查。

### 3. 长期记忆

区别于简单的「带上最近 N 条对话」，系统实现了一个轻量级长期记忆机制：

- 每次新消息后，自动取前 3 个用户问题和前 3 个助手回答的关键内容生成摘要
- 摘要经 embedding 向量化后存入 `ConversationSummary` 表
- 下次提问时，计算问题与历史摘要的余弦相似度，召回相关话题

这使得用户在不同会话间切换话题后，回来时系统仍能感知之前的对话上下文。

## 技术选型

| 组件 | 选型 | 说明 |
|------|------|------|
| 应用框架 | FastAPI | Python 异步 Web 框架 |
| 向量数据库 | Milvus | 分布式向量检索，IVF_FLAT 索引 |
| 嵌入模型 | Qwen3-Embedding-8B | 1024 维，OpenAI 兼容 API |
| 大语言模型 | 可动态切换 | 通过模型配置页面运行时切换，无需重启 |
| 文档解析 | pypdf / pdfplumber / python-docx / pytesseract | 覆盖 PDF/Word 的文本、表格、图片 OCR |
| 前端 | React + Ant Design | SSE 流式处理进度展示 |
